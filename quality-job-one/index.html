Does anyone actually use TDD?

  • I frequently don't know what I want my code to do until I start to write it.
  • But, in general, prefer tests that test functionalty, not just raw input and output
  • Isolate components & systems, not functions and methods
  • Be as terse as possible with your API

Some gripes

  • Don't pollute the global namespace (*ahem* mocha)
  • I want to be able to run the test without the test runner
  • Make it easy to generate coverage reports
