TDD? BDD?

TDD and BDD refer to both methodologies for testing but also specific verbiage in writing tests.

TDD

Test Driven Development talks about suites of tests which expect certain results from calling functons and methods.

TDD also usually means you write tests (which fail) then code (so the tests pass)

BDD

Behavior Driven Development describes what your software should do by describing its features.

BDD usually involves writing code then testing inputs and outputs for components or objects that users would interact with

Unit Testing VS Feature Testing

Unit testing refers to the creation of a test for every single public (and sometimes even private) method or function.

Feature testing refers to the creation of a test for each subsystem, which by virtue tests the other methods made available (just not directly.)
